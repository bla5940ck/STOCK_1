## User Scenarios & Testing *(mandatory)*

### User Story 1 - 取得三大指數摘要 (Priority: P1)

作為 LINE Bot 使用者，我輸入「美股」後，希望立即看到昨晚美股三大重點指數（S&P 500、那斯達克、費城半導體）的漲跌幅，並以中文條列式呈現，讓我快速掌握市場方向。

**Why this priority**: 這是核心入口情境，所有後續查詢與延伸分析都建立在即時掌握市場概況之上。

**Independent Test**: 僅實作此故事即可形成可用 MVP；可透過輸入「美股」驗證是否回傳三大指數、數值正確且格式為中文條列式。

**Acceptance Scenarios**:

1. **Given** 使用者在 LINE 對話中輸入「美股」, **When** 系統完成資料查詢, **Then** 回覆必須包含 S&P 500、那斯達克、費城半導體三者的昨晚漲跌幅。
2. **Given** 查詢成功, **When** Bot 回覆內容, **Then** 回覆必須為繁體中文且採條列式格式呈現。
3. **Given** 其中一項指數暫時無法取得, **When** Bot 回覆結果, **Then** 需清楚標示該項資料暫缺並仍回傳其他可用指數資訊。

---

### User Story 2 - 以代碼查詢個股與相關新聞 (Priority: P1)

作為使用者，我輸入美股代碼（例如 GOOG、TSLA）後，希望取得該代碼前晚漲跌幅與最新相關新聞重點，協助我快速判讀個股動態。

**Why this priority**: 使用者指定代碼查詢是高頻需求，直接影響 Bot 的實用性與黏著度。

**Independent Test**: 可獨立透過輸入有效代碼驗證是否回傳前晚漲跌幅與新聞摘要；透過無效代碼驗證錯誤處理。

**Acceptance Scenarios**:

1. **Given** 使用者輸入有效美股代碼（例如「goog」）, **When** 系統完成查詢, **Then** 回覆必須包含該代碼前晚漲跌幅與至少一則最新相關新聞重點。
2. **Given** 使用者輸入代碼大小寫混用（例如「TsLa」）, **When** 系統解析代碼, **Then** 系統應正確辨識同一檔股票並回傳結果。
3. **Given** 使用者輸入無效或不存在的代碼, **When** 系統查詢失敗, **Then** Bot 應回覆可理解的錯誤訊息與可重試引導。

---

### User Story 3 - 取得美國重要經濟新聞 (Priority: P2)

作為使用者，我輸入「新聞」後，希望看到最新美國重要經濟新聞摘要，以便快速掌握總體市場訊號。

**Why this priority**: 這是重要補充情境，可提升使用者對市場解讀的完整性，但優先度次於指數與個股查詢。

**Independent Test**: 僅實作此故事即可交付單一價值功能；輸入「新聞」即可驗證是否回傳最新重要經濟新聞且為條列式中文。

**Acceptance Scenarios**:

1. **Given** 使用者輸入「新聞」, **When** 系統完成抓取, **Then** 回覆必須包含最新美國重要經濟新聞條列摘要。
2. **Given** 新聞來源回傳多筆資料, **When** Bot 整理內容, **Then** 回覆須以時間與重點排序並避免重複新聞。

---

### User Story 4 - 延伸查詢台股關聯標的 (Priority: P2)

作為使用者，在查詢美股代碼後，我希望可以選擇是否查看台股相關個股，幫助我快速找到與該美股在產業或供應鏈上有關聯的台股標的。

**Why this priority**: 這是差異化功能，可將美股事件連動到在地投資決策，提升產品價值。

**Independent Test**: 在完成個股查詢後，驗證系統是否提供「是否查詢台股關聯股」選項，並在選擇後回傳對應台股清單與關聯說明。

**Acceptance Scenarios**:

1. **Given** 使用者完成某美股代碼查詢, **When** Bot 顯示後續操作, **Then** 必須提供是否查詢台股關聯個股的明確選項。
2. **Given** 使用者選擇查詢台股關聯個股, **When** 系統完成查詢, **Then** 回覆需呈現與該美股相關的台股清單與關聯理由。
3. **Given** 使用者不選擇延伸查詢, **When** 本輪對話結束, **Then** 系統不應主動推送台股資訊。

### Edge Cases

- 使用者僅輸入空白、符號或非股票代碼字串時，系統需提示可用指令範例（如「美股」「新聞」「TSLA」）。
- 同一使用者短時間內連續查詢多檔代碼時，系統應依序回覆，避免內容錯置。
- 新聞來源暫時無新資料時，系統需回覆「目前暫無最新新聞」並提供稍後重試建議。
- 指數資料日期跨時區造成「昨晚 / 前晚」歸屬差異時，回覆需明確附上資料日期。
- 部分美股代碼可能無對應台股關聯標的時，系統需清楚回覆「該代碼暫無相關台股」而非沉默處理。
- 當 API 或資料來源暫時不可用時（超時、網路錯誤），系統回覆清晰的錯誤訊息（「無法連接資料來源，請稍後重試」）並建議使用者重新查詢，不返回過期快取資料。
- 新聞摘要回覆超過 LINE 訊息長度限制時，系統應拆分為多則訊息並自動排序，確保使用者可完整接收所有資訊。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系統 MUST 在使用者輸入「美股」時，提供 S&P 500、那斯達克、費城半導體昨晚漲跌幅。
- **FR-002**: 系統 MUST 支援使用者輸入美股代碼查詢，並回傳該代碼前晚漲跌幅。
- **FR-003**: 系統 MUST 在代碼查詢結果中提供最新相關新聞摘要，每次回覆 3～5 則新聞，每則摘要控制在 100～150 字繁體中文。
- **FR-003a**: 系統 MUST 使用 Alpha Vantage 或 Yahoo Finance API 作為指數、個股、新聞資料的主要來源。
- **FR-004**: 系統 MUST 在使用者輸入「新聞」時，提供最新美國重要經濟新聞摘要，每次回覆 3～5 則，每則 100～150 字。
- **FR-005**: 系統 MUST 將所有回覆以繁體中文條列式輸出。
- **FR-006**: 系統 MUST 在完成美股代碼查詢後提供是否查詢台股關聯個股的選項。
- **FR-007**: 系統 MUST 在使用者同意後，透過實時動態查詢提供與該美股相關的台股個股清單與關聯說明。
- **FR-007a**: 系統 MUST 在代碼查詢完成後，以 LINE 快速回覆按鈕向使用者提供「查詢台股關聯股」（是）與「不查詢」（否）兩個選項，讓使用者可點擊選擇而無需手動輸入。
- **FR-008**: 系統 MUST 對無效輸入或資料取得失敗（如 API 超時、網路異常）提供清晰的錯誤訊息，並建議使用者稍後重試，不主動返回過期快取資料。
- **FR-009**: 系統 MUST 在輸出中標示所使用的資料日期，避免「昨晚 / 前晚」理解歧義。
- **FR-010**: 系統 MUST 對同一代碼的大小寫輸入進行一致辨識（例如 goog 與 GOOG 視為相同查詢）。
- **FR-011**: 系統 MUST 對任何美股代碼查詢，在使用者同意時提供其相關台股清單與關聯說明，無論漲跌幅如何。

### Key Entities *(include if feature involves data)*

- **使用者查詢請求**: 代表使用者一次輸入行為，屬性包含查詢類型（美股摘要 / 代碼 / 新聞）、原始輸入字串、查詢時間、語系。
- **市場指數摘要**: 代表三大指數的單日表現，屬性包含指數名稱、漲跌幅、資料日期、資料可用狀態。
- **個股行情結果**: 代表單一美股代碼查詢結果，屬性包含代碼、公司名稱、前晚漲跌幅、資料日期。
- **新聞項目**: 代表單則新聞摘要，屬性包含標題、摘要（100～150 字繁體中文）、發布時間、來源、關聯代碼或經濟主題；每次查詢回覆 3～5 則，以時間與重點排序。
- **台股關聯標的**: 代表與指定美股存在產業或供應鏈關聯的台股清單，屬性包含台股代碼、名稱、關聯理由、關聯強度；每次查詢時透過實時動態查詢（而非固定對照表）檢索取得。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 95% 以上的「美股」查詢可在 5 秒內回覆完整三大指數條列資訊。
- **SC-002**: 95% 以上的有效代碼查詢可回覆前晚漲跌幅與至少 1 則相關最新新聞。
- **SC-003**: 100% 回覆內容符合繁體中文且為條列式格式。
- **SC-004**: 90% 以上觸發台股延伸查詢的對話可成功完成「是否查詢」選擇流程並回傳結果。
- **SC-005**: 使用者對主要任務（美股摘要、代碼查詢、新聞查詢）首次操作成功率達 90% 以上。
- **SC-006**: 錯誤情境下（無效代碼、資料缺失）100% 提供可重試提示與替代指令建議。

## Assumptions

- 使用者主要為繁體中文讀者，且使用 LINE 作為唯一互動入口。
- 「昨晚」與「前晚」以美股交易日對應之最近兩個已收盤交易日資料為準。
- 美國重要經濟新聞以高影響力總體事件為主（如利率、通膨、就業、GDP、聯準會訊息）。
- 新聞摘要每次回覆 3～5 則最新新聞，每則摘要 100～150 字，以發布時間與重點降冪排序。
- 台股關聯標的透過實時動態查詢（API 或資料庫）取得，而非固定對照表。
- 指數與個股資料主要來源為 Alpha Vantage 或 Yahoo Finance API；若其他 API 更穩定或成本更低可考慮替換，但需保持接口一致。
- 台股查詢交互採用 LINE 快速回覆按鈕，讓使用者可點擊選擇「查詢」或「不查詢」。
- 當資料取得失敗（API 超時、網路異常）時，系統回覆清晰錯誤訊息並建議稍後重試，不返回過期快取資料以避免誤導投資決策。
- 本功能範圍聚焦資訊查詢與摘要，不包含下單、投資建議保證或個人化資產配置建議。
